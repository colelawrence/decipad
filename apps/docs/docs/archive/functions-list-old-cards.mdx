### Round a Number

:::tip Syntax

`round(Number, [Precision])`

:::

Rounds a number to a specified number of decimal places or to the nearest whole number. You can also round a number down with `floor()` or `rounddown()` and round a number up with `ceil()` or `roundup()`.

:::note Examples

`round(3.145)` returns `3`. <br/>
`roundup(3.145, 2)` return `3.15`. <br/>

:::

<br />

### Square Root

:::tip Syntax

`sqrt(Number)`

:::

Calculates the square root of a given number.

:::note Examples

`sqrt(9)` returns the square root of 9, which is `3`.

:::

<br />

### Absolute Number

:::tip Syntax

`abs(Number)` <br/>
`abs(Table.Column)`

:::

Returns the absolute value of a number, removing the negative sign if present. You can also use the `abs()` formula to obtain the absolute values from a column of numbers.

:::note Examples

`abs(-20)` gives you the absolute value of -20, which is 20.

:::

<br />

### Trigonometric Functions

:::tip Syntax

`sin(Angle)` / `asin(Angle)`: Computes the sine and inverse sine of an angle. <br/>
`cos(Angle)` / `acos(Angle)`: Computes the cosine and inverse cosine of an angle. <br/>
`tan(Angle)` / `atan(Angle)`: Computes the tangent and inverse tangent of an angle.<br/>

:::

Trigonometric functions can be used to calculate distances or angles.

:::note Examples

`sin(2 rad)` <br/>
`sin(45 degrees)`

:::

<br />

### Factorial

:::tip Syntax

`factorial(Number)`

:::

Computes factorials, which are commonly used in probability calculations.

:::note Examples

`factorial(3)`

:::

<br />

### Natural Logarithm

:::tip Syntax

`ln(Number)`

:::

Calculates the natural logarithm of a given number.

:::note Examples

`ln(1)`

:::

### Division Remainder

:::tip Syntax

`[Number] mod [Number]`

:::

Calculates the remainder when one number is divided by another.

:::note Examples

`7 mod 3` = `1` <br/>
`10 mod 2` = `2`

:::

<br />

### Get Number Units

:::tip Syntax

`getunit(Number)`

:::

Removes the units from a number, allowing you to work with the numbers alone.

:::note Examples

`stripunit(5$ per mug)` will return `5`.

:::

<br />

### Remove Number Units

:::tip Syntax

`stripunit(Number)`

:::

Retrieves the units of a given number.

:::note Examples

`getunit(10$)` will return `$`.

:::

<br />

### Sum Rows

:::tip Syntax

`sum(Table.Column)` <br/>
`total(Table.Column)`

:::

Calculates the sum of your column.

:::note Examples

`sum(Prices)` <br/>
`sum(Sales.Prices)`

:::

<br />

### Sum Rows that Meet a Condition

:::tip Syntax

`sumif(Table.Column, Table Column Condition)`

:::

Calculates the sum of an entire column based on your specified condition.

:::note Examples

`sumif(Sales.Prices, Sales.Discount > 10) `

:::

<br />

### Column Average

:::tip Syntax
`average(Table.Column)`

:::

Calculate the average of a column. You can also use alternative syntaxes for improved readability: `avg()` and `mean()`.

:::note Examples

`average(Sales.Prices)`

:::

<br />

### Column Average that Meet a Condition

:::tip Syntax

`averageif(Table.Column, Table Column Condition)`

:::

Calculate average of a column of numbers that meet your specified condition. You can also use alternative syntaxes for improved readability: `avgif()` and `meanif()`.

:::note Examples

`average(Sales.Prices, Sales.Discount > 10) `

:::

<br />

### Count Rows

:::tip Syntax

`count(Table.Column)`

:::

Counts the number of rows in a column.

:::note Examples

`count(Sales.Entries)`

:::

<br />

### Count Rows that Meet a Condition

:::tip Syntax

`countif(Table.Column, Condition)`

:::

Counts the number of rows on a column that satisfy the specified condition.

:::note Examples

`countif(Flights.Passengers > 100)`

:::

:::

:::

Extracts the last element from a column.

:::note Examples

:::

### Sort Table Rows

:::tip Syntax

`sort(Table.Column)`

:::

Reorders a column in ascending order.

:::note Examples

`sort(Flight.Prices)`

:::

<br />

### Sort Table Rows by Column

:::tip Syntax

`sortby(Table, Column)`

:::

Reorder table rows based on a specific column.

:::note Examples

### Reverse Table Rows Order

:::tip Syntax

`reverse(Table)`

:::

Reverses the order of rows. Also works with single columns: `reverse(Table.Column)`.

:::note Examples

`reverse(Purchases)` <br/>
`reverse(Purchases.Dates)`

:::

<br />

### Column Size

:::tip Syntax

`len(Table.Column)`

:::

Returns the number of elements in a column.

:::note Examples

`len(Sales.Entries)`

:::

<br />

### Column Max Value

:::tip Syntax

`max(Table.Column)`

:::

Calculate the largest number from a column of numbers or the most recent date from a column of dates.

:::note Examples

`max(Prices.Discount) ` <br/>
`max([date(2050-Jan-01), date(2025-Jun-01)])` <br/>
`max(Sales.Dates)`

:::

<br />

### Column Min Value

:::tip Syntax

`min(Table.Column)`

:::

Calculate the smallest number from a column of numbers or the earliest date from a column of dates.

:::note Examples

`min(Prices.Discount) ` <br/>
`min([date(2050-Jan-01), date(2025-Jun-01)])` <br/>
`min(Sales.Dates)`

:::

<br />

### Unique Values

:::tip Syntax

`unique(Column)`

:::

Extracts the uniques values from a column.

:::note Examples

`unique(Sales.ClientsID)`

:::

<br />

### Forecast Growth

:::tip Syntax

`grow(Initial, Rate, Table.Column)`

:::

Calculates the compound growth of an initial value across your rows.

:::note Examples

`grow($10k, 5%, Investment.Years)`

:::

<br />

### Step Growth

:::tip Syntax

`stepgrowth(Table.Column)`

:::

Calculates the change between values in a column.

:::note Examples

`stepgrowth(SalaryYears.Amount)`

:::

<br />

### Combine Tables

:::tip Syntax

`cat(Table, Table)`

:::

Combines two tables into a new table.

:::note Examples

`cat(SalesDay1, SalesDay2)`

:::

<br />

### Combine Columns

:::tip Syntax

`cat(Table.Column, Table.Column)`

:::

This formula joins two columns (of the same type) together, resulting in a new column. It can also add an element to the end or beginning of a column `cat(Item, Column)`.

:::note Examples

`cat(Day1.Sales, Day2.Sales)` <br/>
`cat(5$, Day2.Sales)`

:::

<br />

### Reference Previous Values

:::tip Syntax

`previous(InitialValue, [Table.Column])`

:::

References the previous value within a table. The first argument is the initial value, and if you want to reference the previous result in another column, you can pass its name as the second argument.

:::note Examples

`previous($10) + 5%` adds `$10` to the first row and adds `5%` to each subsequent row, taking into account the previous value.

### Compound Rate

:::tip Syntax

`compoundrate(Rate, Number of Periods)`

:::

Calculates the compound rate over a specified number of periods based on a constant rate.

:::note Examples

`compoundrate(5%, 36 months)`

:::

<br />

### Future Value

:::tip Syntax

`futurevalue(Rate, Number of Periods, Initial Amount)`

:::

Calculates the future value of an investment considering the total number of periods, initial amount, and a constant interest rate. `fv()` is the compact format for readability.

:::note Examples

`futurevalue(5%, 36 months, $50k)`

:::

<br />

### Net Present Value

:::tip Syntax

`netpresentvalue(Discount Rate, Future Cashflows Column)`

:::

Calculates the net present value of an investment based on a list or column of periodic cash flows and a discount rate. `npv()` is the compact format for readability.

:::note Examples

`netpresentvalue(5%, Table.Column)`

:::

<br />

### Payment Amounts

:::tip Syntax

`paymentamounts(Rate, Number of Periods, Amount)`

:::

Calculates the annual payment amount of a loan considering the borrowed amount, the number of repayment periods, and a constant interest rate. `pmt()` is the compact format for readability.

:::note Examples

`paymentamounts(3%, 36 months, $10k)`

:::

<br />

### Check if a Range Contains a Date

:::tip Syntax

`[Range] contains [Date]`

:::

Checks if a specific date is within a given range.

:::note Examples

`range(date(2050-Jan-01) through date(2050-Dec-31)) contains date(2050-Feb-02 15:30)`

:::

<br />

### Get Year, Month, or Day from Date

:::tip Syntax

`pick(Date, [year/month/day])`

:::

Extracts the year, month, or day from a given date.

:::note Examples

`pick(date(2050-Jan-01), year)`

:::

<br />

### Date Precision

:::tip Syntax

`round(Date, [year/month/day])`

:::

Adjusts the precision of a date. It is useful when you want to perform calculations based on the month or year instead of a specific day.

:::note Examples

`round(date(2050-Jan-01), month)`
